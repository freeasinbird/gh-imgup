# Scope the npm package to @freeasinbird

Branch `chore/package-metadata` (folded into PR #32). The package is published
under the **freeasinbird org**, and npm orgs are scopes — an unscoped
`name: "gh-imgup"` would have published to the global namespace, NOT under the
org. Recommended and chosen: **scoped**.

## Decision: scoped over unscoped-org-owned

Scoped (`@freeasinbird/gh-imgup`) over keeping the bare `gh-imgup` with the org
as owner, because: it matches the stated intent; the org is a real brand likely
to ship more packages (a consistent `@freeasinbird/*` family beats a grab-bag);
and the scope is owned, so no name-squatting race. The only cost is a longer
`npx @freeasinbird/gh-imgup` — and that hits only the npx/install string: the
CLI command stays `gh-imgup` (bin names it explicitly), and the `gh imgup`
extension + skill are unchanged.

## Changes

- `package.json`: `name` → `@freeasinbird/gh-imgup`; add
  `"publishConfig": { "access": "public" }` — **load-bearing**: scoped packages
  publish PRIVATE by default, so the first `npm publish` would fail / go private
  without it.
- Docs: scoped only the **npm-registry invocations** (`npx …`, `npm i -g …`,
  the `@version` pin) in README, AGENTS.md, docs/design.md. Left untouched: the
  `gh-imgup` command itself, `gh extension install freeasinbird/gh-imgup`,
  `gh imgup`, `npx skills add freeasinbird/gh-imgup` (repo paths / bin command,
  not npm). SKILL.md had no npm invocations. Fixed a stale pin example
  (`@1.0.0` → `@0.1.0`) while scoping that line.

## Verification

- `npm pack --dry-run`: name `@freeasinbird/gh-imgup`, tarball
  `freeasinbird-gh-imgup-0.1.0.tgz`, still 12 files; `npm pkg get` confirms
  name + publishConfig. Grep confirms zero remaining unscoped npm invocations
  and the repo/extension refs intact. `npm test` 175/175, lint/typecheck/format
  clean. (No test asserts the package name; `version()` reads `version` only.)
