# Issue 15 cleanup folded into PR 20

Branch `fix/preflight-validation-docs`. The user asked to add the remaining
issue #15 items to PR #20 after confirming overlap: the Markdown extraction and
the remote-parser design-doc refresh (the latter folded into the docs commit of
[[2026-06-25-1134-preflight-validation-docs]]).

## Decisions

- Moved the shared rendered-inline Markdown normalization and alt-text escaping
  into `src/markdown.ts`. This keeps the public comment token guard, cleanup
  reference scan, and upload Markdown formatter on one implementation.
- Kept `decodeMarkdownEntities` and `unescapeMarkdownBackslash` private. The
  repeated shape is the rendered-inline operation, not its internals.
- The extraction is a near-verbatim move (plus the Biome regex tidy that lands on
  `unescapeMarkdownBackslash`), so it is its OWN mechanical commit, recorded in
  `.git-blame-ignore-revs` so blame still points at the original authorship.
- Added `src/markdown.test.ts`: the module now owns security-relevant functions
  (`renderInlineMarkdown` — invariants 3/8; `escapeAltText` — alt-text injection
  guard), so they get direct unit tests rather than only transitive coverage.
- Refreshed the `parseGitRemoteUrl` design note to describe the shipped
  structural parser: URL/scp-like SSH grammar, exact github.com host binding,
  allowed git transports, and userinfo redaction on parse failures.
