# CI hardening: Node matrix, coverage, dependabot, CodeQL

Second PR from the project review (first: security-review fixes). Scope:
CI/supply-chain guardrails only, no product code.

## Decisions

- **Node 22 + 24 matrix.** `engines` says `>=22` but CI only proved 22.
  Branch protection requires the `check` context by name, so the matrix
  reports through a fan-in `check` job with `if: always()` + an explicit
  result test: a plain `needs:` job skips when a leg fails, and GitHub
  treats a skipped required check as satisfied (fails open). Verified the
  required context is exactly `["check"]` via the branch-protection API.
- **Coverage is an informational CI step** (`node --test
  --experimental-test-coverage`), deliberately in ci.yml rather than a
  package.json script so this PR doesn't conflict with the open
  security-fixes PR touching `scripts`; no threshold gate yet (decide one
  after a few runs).
- **dependabot**: npm (devDeps only, invariant 2) + github-actions, weekly.
- **CodeQL** javascript-typescript, `build-mode: none`, on PR/push/weekly
  cron; repo is public so scanning is free. Not added to required checks
  (that's a repo-settings change, maintainer call).

## Deferred / re-deferred

- Promote queue grep: the standing items (sha256 binding, apierr note,
  cleanup pagination) remain covered by invariants 3/4/6 per earlier
  entries; nothing new to promote from this scope. Re-deferred.
- OS matrix (macOS/Windows) deferred: no platform-specific code paths
  beyond `rm -rf` in a dev-only script; add if a platform bug ever shows.
- Coverage threshold gate deferred until baseline numbers exist.
